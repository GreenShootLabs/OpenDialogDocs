Conversational Applications

Conversational applications

OpenDialog is a framework for building and managing conversational applications.

A conversational application is an application whose main mode of interaction with the user is through the exchange of messages, which we call utterances. Each utterance carries an intent. An intent describes the underlying purpose of the party sending the utterance and that purpose can be expressed using any number of different utterances.

Difference between utterances and intents

An utterance is when a user says something like:

- Please create a new task checklist.

The intent of this utterance is to command an application to create a new task checklist.

The user could have also said

- Create me a new list

or

- Start a checklist

or

- Get a checklist going

All these different utterances can map to the same intent and it's

We refer to the task of mapping an utterance to an intent as intent interpretation.

The conversational engine at the core of OpenDialog provides a way to describe what exchange of intents a specific conversation considers as useful or valid and provides tools to hook into intent interpreters so as to map utterances to those intents and also a structure through which to perform actions (e.g. actually create a checklist).

Why do you call them conversational applications and not just chatbots?

We use the term conversation application rather than chatbot because chatbot as a term is limiting in explaining the full scope of what OpenDialog attempts to cover.

OpenDialog doesn't just deal with the conversational aspect, but the entire application. A chatbot, in the abstract, cannot achieve anything. It needs to be able to plug in to other applications or APIs (or just stuff in general) in order to get things done. OpenDialog provides ways to do that in a consistent manner so as to build the entire application and not just the conversational aspect of it.

Conversations

At the core of a conversational application is obviously the conversation itself. An exchange of messages - utterances - between the user and the application.

The question we face as designers and builders of conversational application is how do we represent these conversations. What conceptual model do we use to think about the exchange of utterances between users?

Multi-agent systems research has dealt with these problems for decades. Multi-agent systems deal with the issues that arise when there are multiple problem-solving entities (in our case humans and bots) in a constnatly changing context with multiple perspectives.

OpenDialog is inspired by work coming from the multi-agent systems research space and, in particular, the use of electronic institutions and enhanced Dooley graphs to represent the exchange of messages between participants in a conversation.
